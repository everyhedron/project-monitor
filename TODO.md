- [x] we are still getting duplicated project monitor pins on startup. can you check how agent monitor solved this issue and implement the fix?
  > Response: Checked — agent-monitor's `extension.ts` has the exact same pattern (`if (!dashboard.hasPanel()) dashboard.open(...)` inside a 1s `setTimeout`), so it hasn't actually solved this; it likely has the same bug. The real race: on window restart, VS Code restores a previously-open Project Monitor tab via the registered `WebviewPanelSerializer`, but that restoration is async and isn't guaranteed to finish before the `openOnStartup` timeout fires — if it hasn't, `hasPanel()` is still false, so a second panel gets opened, and then the serializer's restore call lands afterward as a duplicate. Fixed by also checking `vscode.window.tabGroups.all` for an existing webview tab of our view type before opening — that tab placeholder exists as soon as the window layout is restored, before the panel content itself finishes deserializing, so it's not subject to the same race.

  > Response: Yes. `openProjectAgent` (now the "codex" button) copies the project path to the clipboard, then checks for an already-open terminal named `"<Project Name> | Codex"` and reveals it if found.
